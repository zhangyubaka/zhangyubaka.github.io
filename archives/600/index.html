6in4 隧道在 OpenWRT 上的配置

Jun 21, 2016

he.net 的隧道应该都知道吧?不知道的话,去 https://tunnelbroker.net/ 申请一个.

他的隧道走的是 6in4,需求嘛,和 isatap, 6rd, 6to4 差不多,需要打开 41 协议(不是端口).这里就在 OpenWRT 上配置.

在下的路由的固件比较老了,用的还是 radvd 来广播,新的固件中 odhcpd 会直接广播.

首先,选择 wan6,协议选择 IPv6-in-IPv4,你可能需要安装 6in4, opkg install 6in4.

切换协议之后你看到的大概是这样.

其中,本地 IPv4 地址留空就好了,本地 IPv6 地址则填入 Client IPv6 Address: 后面的那个,远程 IPv4 地址填 Server IPv4 Address,很明显,很简单.IPv6 routed prefix 则是 Routed IPv6 Prefixes 下面的,填 /64 或 /48 随你便咯.

然后保存,连接一下试一试,是不是可以了?

然后呢,配置一个 crontab 来更新你的 ip 地址,因为 Luci 上面那个不知道为什么不能用.

#crontab -e
wget -q -O "http://username:Update Key@ipv4.tunnelbroker.net/nic/update?hostname=tunnelid"

后面那个链接在 HE 的界面中的 Advanced 区有, Example Update URL 那里,抄下来就好了.

还有个问题是分配 ipv6 地址,如果你是最新版的,那就没什么问题,老版的话则是配置一下 radvd,在 Luci 设置一下就好了.
