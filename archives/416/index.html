Gentoo BINHOST

Apr 21, 2016

Hmm,请不要在意为什么 BINHOST 是全大写的.

BINHOST,或者 Binary Host,就是所谓 二进制分发服务器 了,具体就是跟那些 rpm 源啊,apt 源啊一样,可以让你省去编译的苦恼.

不过你不编译为什么要用 Gentoo?

我找到了 Nixbits 的 BINHOST,而且他看起来也有些年头了,构建的平台分别是 Atom Nocona Celeron

这里的 Celeron 和 Nocona 分别是 Celeron M 和 Socket 604 的 Xeon…估计…可以用吧.

首先在 make.conf 里面加入四个地址中的一个.

#vim /etc/portage/make.conf

PORTAGE_BINHOST="http://gentoo.nixbits.net/binhost/no-x-x86_64-atom/    #无 X 和 GTK 支援, Atom 构建.

PORTAGE_BINHOST="http://gentoo.nixbits.net/binhost/x-gtk-x86/    #有 X 与 GTK 支援,32 位元, Celeron 构建.

PORTAGE_BINHOST="http://gentoo.nixbits.net/binhost/x-gtk-x86_64-atom/    #有 X 与 GTK 支援, Atom 构建.

PORTAGE_BINHOST="http://gentoo.nixbits.net/binhost/x-gtk-x86_64-nocona/    #有 X 与 GTK 支援, Nocona 构建.

PS:千万别都加进去了

这里也有他的 Rsync 地址:

rsync://gentoo.nixbits.net/gentoo-binhost
rsync://gentoo.nixbits.net/gentoo-portage
